# Broker Chat — UI 界面指南

本文档详细介绍 Broker Chat VS Code 扩展的界面布局与各功能区域。

---

## 整体风格

Broker Chat 采用深色科技风设计：

- **深蓝黑背景**（`#0d1117`），带微弱的青色（Codex 色）和橙色（Claude 色）径向渐变
- **毛玻璃质感**顶栏，带 `backdrop-filter: blur` 效果
- **圆角卡片**（16-18px radius），带半透明面板和柔和阴影
- **品牌色**：
  - Codex = `#23c7b6`（青绿色）
  - Claude = `#f29f43`（橙色）
  - User = `#7aa2ff`（蓝色）
  - Thinking = `#8b9bb7`（灰蓝）

## 各区域详解

### 1. 顶部栏（Topbar）

固定在侧边栏顶部，包含：

| 元素 | 说明 |
|------|------|
| 品牌标题 | "Broker Chat"，17px 加粗 |
| 副标题 | "官方会话监控与桥接"，12px 灰色 |
| 设置按钮 ⚙ | 切换设置面板的展开/折叠 |
| 刷新按钮 🔄 | 手动刷新监控数据 |

背景使用毛玻璃效果（`backdrop-filter: blur(14px)`），底部有 1px 边框分隔线。

### 2. 状态栏（Status Bar）

4 个圆角胶囊（pill），水平排列，自动换行：

| 胶囊 | 含义 | 状态点颜色 |
|------|------|-----------|
| **项目** | 当前工作区名称 | — |
| **监控** | 会话监控状态 | 🟢 运行中 / ⚫ 已关闭 |
| **桥接** | 桥接发送状态 | 🟢 待命 / 🟡 发送中 / 🟢 完成 / 🔴 失败 |
| **自动转发** | 自动转发状态 | ⚫ 关闭 / 🟢 待命 / 🟡 等待 / 🟡 发送 / 🟢 完成 / 🔴 失败 |

### 3. 设置面板（Settings Panel）

点击 ⚙ 按钮展开，再次点击折叠。包含：

- **自动转发开关**：复选框 + 标签
- **关键词编辑区**：左右两列文本框
  - 左列："发给 Codex 关键词"（如 `给Codex命令`、`问codex`）
  - 右列："发给 Claude 关键词"（如 `给Claude命令`、`问claude`）
- **操作按钮**：
  - 保存关键词 — 保存自定义关键词
  - 恢复默认 — 重置为内置默认关键词

### 4. 连接验证面板（Verify Panel）

可折叠区域（`<details>` 元素），点击箭头展开/折叠：

**折叠状态**显示：
- 面板标题 + 右侧折叠箭头（▶ / ▼）

**展开状态**显示：
- **会话卡片**（两列并排）：
  - Codex Session 卡片
  - Claude Session 卡片
  - 每张卡片包含：
    - 会话标题（用户的首条问题）
    - 会话 ID
    - 消息数量
    - 最后更新时间（相对时间，如 "3 分钟前"）
    - 来源文件路径（悬停显示完整路径）
    - CWD（工作区目录）

- **最近消息预览**（两列并排）：
  - 左列：Latest Codex 预览（最多 4 条）
  - 右列：Latest Claude 预览（最多 4 条）

### 5. 合并时间线（Merged Timeline）

核心功能区域，展示所有消息按时间排列：

**消息卡片**特征：

- 左侧彩色竖线标识来源角色：
  - 🔵 蓝色 = 用户消息（`border-left: 4px solid #7aa2ff`）
  - 🟢 青绿 = Codex 回复（`border-left: 4px solid #23c7b6`）
  - 🟠 橙色 = Claude 回复（`border-left: 4px solid #f29f43`）

- 消息头部（`message-header`）：
  - 角色徽章（USER / CODEX / CLAUDE）
  - 来源标签（合并时间线中显示 "CODEX" 或 "CLAUDE" 芯片）
  - 时间戳

- 思考过程（Thinking Block）：
  - 可折叠区域，点击 "THINKING" 展开
  - 灰蓝色背景，带独立边框
  - 显示模型的思考过程文本

- 消息正文（`message-text`）：
  - 等宽字体（Consolas）
  - 保留换行和空格（`white-space: pre-wrap`）
  - 支持文本选择和复制

- 操作按钮行（`action-row`）：
  - **合并转发**：带来源色的主按钮（青绿边框 + 半透明背景）
  - **仅转发回答**：灰色次要按钮
  - 按钮文本根据消息来源动态显示（如 Codex 消息显示 "合并转发到 Claude"）

- 补充说明框（`bridge-note-input`）：
  - 可选文本域，用于附加说明
  - 聚焦时显示蓝色边框

## 响应式布局

在窄屏（< 960px）下：
- 关键词编辑区变为单列
- 会话卡片变为单列
- 消息预览变为单列
- 合并消息流取消高度限制

## 交互流程示意

### 手动转发流程

```
用户在 Codex 中提问
       ↓
Broker Chat 监控到消息（1.5s 内）
       ↓
侧边栏合并时间线出现 Codex 回复
       ↓
用户点击 "合并转发到 Claude"
       ↓
（可选）填写补充说明
       ↓
插件自动：写入剪贴板 → 聚焦 Claude 面板 → Ctrl+V → 提交
       ↓
Claude 面板中出现转发的消息
       ↓
Broker Chat 监控到 Claude 的回复
       ↓
合并时间线更新，显示 Claude 回复
```

### 自动转发流程

```
用户在官方插件输入 "给Codex命令：帮我分析这段代码"
       ↓
Broker Chat 监控到消息
       ↓
自动转发引擎匹配关键词 "给Codex命令"
       ↓
等待 Claude 回复完成（3 次轮询确认稳定）
       ↓
自动执行桥接：剪贴板 → 聚焦 → 粘贴 → 提交
       ↓
状态胶囊显示 "已完成" 🟢
```
